dic_attribute_type
The attribute type dictionary table. Contains information about the attribute types.
gid serial NOT NULL Unique attribute type identifier
code varchar( 254 ) Code of the attribute type
description varchar( 254 ) Short textual description of the attribute type
extended_description varchar( 1024 ) Extended textual description of the attribute type
taxonomy_code varchar( 254 ) Code of the taxonomy
attribute_level smallint Identifier of the attribute level (e.g. GEM taxonomy: 1 = main attribute, 2 = secondary attribute, 3 = tertiary attribute)
attribute_order smallint Order of the attribute type. To be used for compiling a textual representation of the taxonomy attributes and their values which follows a predefined order (e.g. GEM Taxonomy TaxT strings)
Indexes
pk_dic_attribute_type primary key ON gid
idx_dic_attribute_type unique ON code
idx_dic_attribute_type_0 ON taxonomy_code
Foreign Keys
fk_dic_attribute_type ( taxonomy_code ) ref dic_taxonomy (code)

logged_actions
History of transactions on activated tables, from history.if_modified_func().
gid bigserial NOT NULL Unique log identifier
schema_name text NOT NULL Textual reference to the database schema which contains the modified table
table_name text NOT NULL Name of the modified table
table_id oid( 10 ) NOT NULL OID of the modified table
transaction_id bigint Identifier of the transaction (may differ from gid when more than one row is affected by a transaction query)
transaction_user text Session user name who caused the transaction
transaction_time timestamptz NOT NULL Timestamp when transaction was started (current_timestamp)
transaction_query text Transaction query
transaction_type text NOT NULL Transaction type (I = insert, D = delete, U = update, T = truncate)
old_record hstore( 2147483647 ) The old record before the modification containing all the values as hstore (for DELETE and UPDATE statements)
new_record hstore( 2147483647 ) The new record after the modification containing all the values as hstore (for INSERT and UPDATE statements)
changed_fields hstore( 2147483647 ) The modified fields only, including the new values, stored as hstore
Indexes
logged_actions_pkey primary key ON gid
logged_changes_action_idx ON transaction_type
logged_changes_table_id_idx ON table_id
Constraints
logged_actions_transaction_type_check transaction_type = ANY (ARRAY['I'::text, 'D'::text, 'U'::text, 'T'::text])


main
The main object table for resolution 1 (e.g. per building scale). Contains basic information about the object.
gid serial NOT NULL Unique object identifier
survey_gid integer Identifier for the survey
description varchar( 254 ) Textual description of the object
source text Source of the object (geometry)
accuracy numeric Accuracy of the object (geometry)
res2_id integer gid of the object at resolution level 2 (e.g. neighbourhood scale)
res3_id integer gid of the object at resolution level 3 (e.g. settlement scale)
the_geom geometry( 2147483647 ) Spatial reference and geometry information
Indexes
pk_main_0 primary key ON gid


main_detail
The main object detail table. Contains information about the object details.
gid serial NOT NULL Unique object detail identifier
object_id integer Object identifier
attribute_type_code varchar( 254 ) Code of the taxonomy attribute type
attribute_value varchar( 254 ) Value of the taxonomy attribute type (from look up table in taxonomy scheme)
attribute_numeric_1 numeric Value of the taxonomy attribute type (numeric)
attribute_numeric_2 numeric Value of the taxonomy attribute type (numeric)
attribute_text_1 varchar( 254 ) Value of the taxonomy attribute type (textual)
Indexes
pk_object primary key ON gid
idx_main ON attribute_value
idx_main_detail ON object_id
pk_main ON attribute_type_code
Foreign Keys
fk_object_gid ( object_id ) ref main (gid)
fk_attribute_type ( attribute_type_code ) ref dic_attribute_type (code)
fk_attribute_value ( attribute_value ) ref dic_attribute_value (attribute_value)

View ve_resolution1
SELECT a.gid, a.survey_gid, a.description, a.source, a.accuracy, a.res2_id, a.res3_id, a.the_geom, b.object_id, b.attribute_type_code, b.attribute_value, b.attribute_numeric_1, b.attribute_numeric_2, b.attribute_text_1, c.detail_id, c.qualifier_type_code, c.qualifier_value, c.qualifier_numeric_1, c.qualifier_text_1, c.qualifier_timestamp_1 FROM ((object_res1.main a JOIN object_res1.main_detail b ON ((a.gid = b.object_id))) JOIN object_res1.main_detail_qualifier c ON ((b.gid = c.detail_id))) ORDER BY a.gid;
